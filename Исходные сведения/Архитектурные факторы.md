## Функциональные требования

- **F01**: Пользователи должны иметь возможность регистрации и авторизации.
- **F02**: Пользователи должны иметь возможность редактировать свой профиль.
- **F03**: Пользователи должны иметь возможность загрузить свои собственные функции в приложение
- **F04**: Пользователь должен иметь возможность запустить свои загруженные функции и получить результат выполнения
- **F05**: Пользователь должен иметь возможность просмотреть список загруженных функций, изменить их параметры и удалить их при необходимости
- **F06**: Пользователь должен иметь возможность посмотреть публичные функции других пользователей
- **F08**: Пользователь должен иметь возможность добавлять/удалять публичные функции в свой список подписок
- **F09**: Пользователь должен иметь возможность посмотреть списка публичные функции на которые он подписан
- **F10**: Пользователь должен иметь возможность объединять триггеры и свои функции
- **F11**: Пользователь должен иметь возможность объединять триггеры и функции на которые он подписан
- **F12**: Пользователь должен иметь возможность просматривать статистику по тому сколько раз была вызвана функция

## Нефункциональные требования

### Производительность (Performance)

- **PER01**: Приложение должно быть отзывчивым и иметь быстрое время отклика
- **PER02**: Приложение должно отображать списочные данные с фильтром за 2 секунды или менее.
- **PER03**: Триггер не должен отрабатывать дольше 7 секунд
- **PER04**: Функция вызванная триггером не должна отрабатывать дольше 10 минут и потреблять больше 5 Гб памяти за один раз
- **PER05**: При множественном вызове функции не должно создаваться больше 10 контейнеров

### Масштабируемость (Scalability)

- **SCA01**: Приложение должно быть способно масштабироваться для работы с большим количеством пользователей
### Доступность (Availability)

- **AVA01**: Допустимый процент отказа или обслуживания системы составляет 99.9%.

### Надежность (Durability, Disaster Recovery)

- **DUR01**: Автоматическое восстановление после сбоев.

### Мониторинг (Monitoring)

- **MON01**: Использование стандартных инструментов мониторинга организации.

### Организационные и юридические вопросы (Legal, Regulatory and Compliance)

- **LRC01**: Соблюдение GDPR при обработке данных европейских пользователей.

### Целевые платформа и устройства пользователей

- **PLT01**: Поддержка последних версий браузеров Chrome, Firefox, Safari, Edge.

## Ограничения

- **LIM01**: Поддержка только двух языков (Русский и Английский).
- **LIM02**: Работа на территории СНГ.

## Принципы

- **PRN01**: Прозрачность данных для пользователей.
- **PRN02**: Стремление к максимальной автоматизации процессов.
- **PRN03**: Код должен быть организован в модули.
- **PRN04**: Возможность добавления новых функциональности и изменения существующей функциональности без полной переработки приложения.
- **PRN05**: Приложение должно быть разработано с учетом возможности проведения тестирования, чтобы обеспечить его надежность и функциональность.
