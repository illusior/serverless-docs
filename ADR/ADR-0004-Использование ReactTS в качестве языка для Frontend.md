# Использование ReactTS в качестве языка для Frontend

Дата: 2023-10-01

## Статус

Принято

## Контекст

Наша задача реалезовать serverless приложение, которое бы могло запускать чужие функции на нашем железе
Возникла необходимость в создании красивого и удобного интерфейса для взаимодействия с нашей системой.

### Варианты

1. react js. это бесплатная библиотека с открытым исходным кодом на JavaScript для построения пользовательских интерфейсов на основе компонентов. Она поддерживается компанией Meta (ранее Facebook) и сообществом индивидуальных разработчиков и компаний. React может использоваться для разработки одностраничных, мобильных или серверных рендеринговых приложений с использованием таких фреймворков, как Next.js.
2. react ts. это "библиотека JavaScript для создания пользовательских интерфейсов", а TypeScript - это "типизированный супернабор JavaScript, который компилируется в обычный JavaScript". Используя их вместе, мы, по сути, создаем пользовательские интерфейсы на основе типизированной версии JavaScript. Причина, по которой вы можете использовать их вместе, заключается в том, чтобы получить преимущества статически типизированного языка (TypeScript) для вашего пользовательского интерфейса. Это означает большую безопасность и меньшее количество ошибок, передаваемых на передний план.
3. Angular. это JavaScript-фреймворк с открытым исходным кодом, предназначенный для разработки одностраничных приложений.
Его цель — расширение браузерных приложений на основе MVC-шаблона, а также упрощение тестирования и разработки. Фреймворк работает с HTML, содержащим дополнительные пользовательские атрибуты, которые описываются директивами, и связывает ввод или вывод области страницы с моделью, представляющей собой обычные переменные JavaScript.

## Решение

Выбранный вариант: "[react ts]", потому что В нашей команде большая часть людей знает именно его, что в свою очередь является большим плюсом для нас, так как не будет необходимости в изучении другого фреймворка. А типизация поможет легче общаться с бэкендом и увеличит безопастность при передачи данных.
